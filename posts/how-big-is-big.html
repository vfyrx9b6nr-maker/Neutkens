How Big is Big?

17 May 2026

A journey to the boundary of the finite, but never beyond.

The Ladder of Understanding: From Billion to Inconceivable

Let's begin with the graspable. A billion seconds is about 31.7 years. A trillion seconds is 31,700 years, older than the last ice age. The numbers we are about to discuss make these examples laughably small. They reduce a trillion to a rounding error. To name these numbers, we almost blow up our very tools for working with numbers. We climb the Fast Growing Hierarchy (FGH).

The Fast Growing Hierarchy: A Ladder of Ever Larger Steps

The Fast Growing Hierarchy (FGH) is a system for indexing functions by their growth rate. It begins simply and then explodes. We build it step by step, with a fixed input n = 3 to keep it manageable. It is instructive to write out each step yourself; only then do you see how rapidly the output grows.

The base. We define the successor function:

f0(n) = n + 1

For n = 3 that is simply 4. This is very special because there is always a succesor. The next step is defined by applying the current step a number of times equal to the input itself:

f1(n) = f0n(n) = n + n = 2n

With n = 3: f1(3) = f0(f0(f0(3))) = f0(f0(4)) = f0(5) = 6.

The next step repeats the current one n times:

f2(n) = f1n(n) = n × 2n

With n = 3: f2(3) = f1(f1(f1(3))) = f1(f1(6)) = f1(12) = 24. Exponential growth.

And so it continues. f3(n) amounts to a tower of powers, n steps high. f3(3) = f2(f2(f2(3))). We know f2(3) = 24, so f2(24) = 24 × 224. That is 402653184, so f2(402653184) = 402653184 × 2402653184. The number of atoms in the universe is about 1080. Perhaps, the number of Wolfram space-atoms is 10358. But f3(n) is a number with about a hundred million digits. It's comparable to the number of possible bookshelves (assuming 100 books, 250 pages, and 100 symbols).

This pattern works for every index, causing the numbers to explode tremendously. But then we make the index itself a large number. We define:

fω(n) = fn(n)

Here something fundamental happens: we needed a new symbol. At first we try to reduce the number of symbols for a large number, but is that really possible for large numbers? We apply the n-th function to n itself. For n = 3:

fω(3) = f3(3)

But we can go further. Here we must be ever more careful to define everything properly, for that ultimately determines our human limit. The index 1 + ω means: first substitute ω and then add one, but the index ω + 1 means: repeat the ω-function:

fω+1(n) = fωn(n)

For n = 3:

fω+1(3) = fω(fω(fω(3)))

We have fω(3) = f3(3), which we already calculated and it was a gigantic number, let's call it G. The next step is fω(G). But note: fω(G) = fG(G). That means we apply the G-th function in the hierarchy to G itself. G was already gigantic, and now we use G as an index in the hierarchy. The final step is fω(fG(G)), which uses the output of the previous step as an index yet again. The mind stutters.

This pattern repeats itself. We race through to ωω, and ultimately to the ordinal number ε0. ε0 is the smallest ordinal for which ωε0 ≈ ε0 (yes, it is that bizarre). The function fε0(n) explodes in a way we can try to imagine... or can we? The amounth fε0(3) is astronomical. Let's try to express it:

The Tree of Trees: TREE(3)

If fε1(3) is an atom, then TREE(3) is the vast universe surrounding it. Surprisingly, the description for TREE(n) is simple: you draw trees (networks of points and lines) with seeds of at most n different colors. Each new tree may not contain a copy of any earlier tree, where 'copy' means that one tree can be embedded into the other via a structure-preserving process. The goal is to keep going as long as possible. The question is: how long can you keep this up before it becomes impossible to draw a new, unique tree?

The answer for n=1 is 1. For n=2 it is 3. For n=3 the answer is TREE(3). And TREE(3) is a number so immensely large that it is 'probably' around fSVO(3). The SVO is a special ordinal that soars far above ω and εεε0. TREE(3) is a finite, exactly defined number. And of course, we can easily define TREE(TREE(3)) or TREETREE(3)(3) as well.

The 'Busy Beaver'

The Busy Beaver game revolves around Turing machines—simple, theoretical computers with a finite number of states. For a given number of states n, there are finitely many machines. Some stop after a while, others run forever. The Busy Beaver function, BB(n), is the maximum number of steps that a halting Turing machine with n states takes. It is the maximum lifespan of simple programs that are not infinite.

For n=4 and n=5, BB(n) is computable. For n=6 it is known that the value is larger than f4(5). For n=7 it may still be expressible in terms of the FGH, but beyond that? With some space on a computer we can easily program the FGH, make games like TREE, and invent so many other tricks. While the BB description is short and valid, we didn't specify the algorithm. In a sense, we are entering the realm of "specifying" infinity as |1/0|.

The number of possible Turing machines for an arbitrary n-state machine with 2 symbols can be calculated with the following formula: N = (4n + 4)2n. For a machine with n = 10 states, this means there are 4420 unique programs (and thus Turing machines). But to check which one stops last, we must wait until every program that will ever stop has stopped. However, there will be programs among them that do not stop but also do not fall into a loop. They continually create new information and no one knows whether they will ever fall silent as a result.

And here the circle closes with our cathedral. The Busy Beaver function for arbitrary n is uncomputable. The universe, driven by a simple, reliable rule, produces causality. But because of that causality, the complexity of reality keeps increasing. This is good news for happiness: we will never be bored, while we were born into a relatively comprehensible situation.

Conclusion: The Sublime Number?

We began with the graspable: 31.7 years of a human life. We end with numbers that resemble infinity more than they do 1, 2, 3, and 4. But TREE(3) and BB(100) are not infinity. They are pure manifestations of the finite. This shows that you need no mysticism or paradox to gaze upon the sublime. A finite, well-defined number can be larger than large. It opens a world we will not soon see, but which we will one day reach. That is the true grandeur of constructivism.
